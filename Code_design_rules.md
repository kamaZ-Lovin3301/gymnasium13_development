Правила оформления кода.
1. Использоваемый язык - Python. Djungo.
2. Использовать стандарт оформления PEP8.
   Отступы: Используйте 4 пробела для каждого уровня отступа. Не смешивайте табуляцию и пробелы.
   Пустые строки:
   -Две пустые строки между функциями и классами верхнего уровня.
   -Одна пустая строка между методами внутри класса.
   Импорты (Imports): Импорты должны быть в начале файла, каждый с новой строки.
   Пробелы: Избегайте лишних пробелов внутри скобок, перед запятой, точкой с запятой или двоеточием.
   Вокруг бинарных операторов (например, =, +, -) должен стоять один пробел, кроме аргументов по умолчанию.
   Именование: Переменные/функции: snake_case (маленькие буквы с подчеркиванием).
   Классы: CamelCase (или PascalCase).
   Константы: UPPER_CASE (заглавные буквы с подчеркиванием).
   Комментарии: Должны обновляться вместе с кодом.
3.  Обязательно создание технической документации в Wiki к реализованным задачам.
4.  Название веток должно быть информативным.
